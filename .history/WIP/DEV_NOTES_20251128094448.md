## FILE ENCODING RULES FOR LRS2-DEV PROJECT

When editing HTML/JS files in this project:

1. **USE ASCII ONLY** - Never use special Unicode characters:
   - Use `-` (hyphen) NOT `—` (em-dash) or `–` (en-dash)
   - Use `...` NOT `…` (ellipsis)
   - Use `*` or `-` NOT `•` (bullet)
   - Use `'` and `"` NOT curly quotes `'` `"` `'` `"`

2. **AVOID TEMPLATE LITERALS FOR USER-FACING TEXT** - When building strings that display to users (labels, activity feeds, dropdowns), use string concatenation:
```javascript
   // GOOD
   const label = poRef + ' - ' + registration + ' - ' + date;
   
   // RISKY (can introduce encoding issues)
   const label = `${poRef} — ${registration} — ${date}`;
```

3. **BEFORE DELIVERING ANY FILE**, run this check:
```bash
   grep -P '[\x80-\xFF]' filename.html | head -10
```
   If it returns anything other than intentional UTF-8 (like emoji), fix it.

4. **IF EDITING A FILE WITH EXISTING CORRUPTION**, clean it first:
```bash
   sed -i "s/\xc2\x9d//g" filename.html
```

The project files have had UTF-8 encoding corruption issues. Be defensive.
